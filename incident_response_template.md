# Basic Incident Response Template

## 1. Identification
- What happened?
- Who reported it?
- When was it first observed?
- What systems or users are affected?

## 2. Containment
- Isolate affected systems
- Block malicious IPs, domains, or accounts
- Preserve logs and evidence

## 3. Eradication
- Remove malware or unauthorized access
- Patch vulnerabilities
- Disable persistence mechanisms
- Reset compromised credentials

## 4. Recovery
- Restore from clean backups if needed
- Return systems to service in stages
- Monitor closely for reinfection or repeated abuse

## 5. Lessons Learned
- Document root cause
- Record timeline and impact
- Improve controls, detections, and user awareness
